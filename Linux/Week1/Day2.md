# 📘 Day 2 - Boot Process · FHS (부팅 과정 · 파일 시스템 계층 구조)

> **부팅 과정(Boot Process)**과 **FHS(File System Hierarchy Standard)**를 이해한다

---

# 1️⃣ 핵심 개념

| 개념 | 영어 | 한 줄 정의 |
|------|------|-----------|
| **부팅 과정** | Boot Process | 전원 ON부터 로그인까지 운영체제가 실행되는 단계별 초기화 과정 |
| **파일 시스템 계층 구조** | File System Hierarchy Standard (FHS) | 리눅스 디렉토리 구조와 각 디렉토리의 역할을 표준화한 규칙 |

---

# 2️⃣ 시스템 동작 구조

## ① Boot Process (부팅 5단계)

```text
┌──────────────┐
│ BIOS / UEFI  │
│ 하드웨어 검사 │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ Bootloader   │
│    (GRUB)    │
└──────┬───────┘
       │ 커널 로딩
       ▼
┌──────────────┐
│    Kernel    │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ init/systemd │
│ 서비스 시작   │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ Login Prompt │
└──────────────┘
```

### 단계별 역할

| 단계 | 역할 |
|------|------|
| **BIOS / UEFI** | CPU, RAM, 디스크 등 하드웨어가 정상인지 검사(POST) |
| **Bootloader (GRUB)** | 실행할 커널을 선택하고 메모리에 적재 |
| **Kernel** | 하드웨어를 제어하며 운영체제를 시작 |
| **init/systemd** | 네트워크, SSH 등 각종 서비스를 실행(PID 1) |
| **Login Prompt** | 사용자가 로그인 가능한 상태 |

> BIOS/UEFI와 Bootloader 단계는 아직 리눅스가 실행된 상태가 아니다.

---

## ② FHS (File System Hierarchy)

```text
/
├── /bin   → 필수 실행 파일
├── /etc   → 설정 파일
├── /var   → 로그, 캐시 등 변경되는 데이터
├── /home  → 사용자 홈 디렉토리
├── /usr   → 프로그램 및 라이브러리
└── /tmp   → 임시 파일
```

### `/etc`와 `/var`의 차이

| `/etc` | `/var` |
|---------|---------|
| 정적(Static) 데이터 | 동적(Dynamic) 데이터 |
| 사람이 수정하는 설정 파일 | 시스템이 계속 생성하는 데이터 |
| 거의 용량이 증가하지 않음 | 시간이 지날수록 커질 수 있음 |

# 3️⃣ 실습

## ① 루트 디렉토리 구조 확인

```bash
ls -l /
```

리눅스는 **FHS 규칙**에 따라 디렉토리를 미리 구성해 둔다.

---

## ② 부팅 시 자동 마운트 정보 확인

```bash
cat /etc/fstab
```

`/etc/fstab`에는 부팅 시 어떤 디스크를 어디에 마운트할지 저장되어 있다.

---

## ③ 부팅 시간 확인

```bash
systemd-analyze
```

| 항목 | 의미 |
|------|------|
| **kernel** | 커널이 하드웨어를 초기화하는 시간 |
| **userspace** | systemd가 각종 서비스를 실행하는 시간 |

---

# 💡 한 줄 요약

> **컴퓨터는 BIOS/UEFI → Bootloader → Kernel → systemd → Login 순서로 부팅되며, 리눅스는 FHS 규칙에 따라 `/etc`에는 설정 파일을, `/var`에는 계속 변경되는 데이터를 저장한다.**
