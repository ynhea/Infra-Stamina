> 
> **Kernel, Shell, Distribution**의 역할과 서로의 관계

---

# 1️⃣ 핵심 개념

| 개념 | 영어 | 한 줄 정의 |
|------|------|-----------|
| **커널** | Kernel | 하드웨어를 직접 제어하고 시스템 자원을 관리하는 핵심 프로그램 (좁은 의미의 Linux) |
| **쉘** | Shell | 사용자와 커널 사이를 연결하는 명령어 해석기 |
| **배포판** | Distribution (Distro) | **커널 + 패키지 관리자 + 기본 도구**를 하나로 묶어 배포한 완성된 운영체제 |

---

# 2️⃣ 시스템 동작 구조

```text
┌─────────────────────────────────────┐
│            사용자 (User)             │
└──────────────┬──────────────────────┘
               │ 명령 입력
               ▼
┌─────────────────────────────────────┐
│        Shell (bash, zsh, sh)         │
└──────────────┬──────────────────────┘
               │ System Call
               ▼
┌─────────────────────────────────────┐
│              Kernel                  │
│                                     │
│ • Process 관리                      │
│ • Memory 관리                       │
│ • Device 관리                       │
│ • File System 관리                  │
└──────────────┬──────────────────────┘
               │ 하드웨어 직접 제어
               ▼
┌─────────────────────────────────────┐
│ Hardware (CPU · RAM · Disk · NIC)   │
└─────────────────────────────────────┘
```

# 3️⃣ 실습

## ① 커널 버전 확인

```bash
uname -r
```

결과

```text
6.8.0-45-generic
```

### 왜 이렇게 되는가?

**커널이 System Call에 직접 응답하여** 현재 실행 중인 Kernel Release를 알려줌

---

## ② 현재 Shell 확인

```bash
echo $SHELL
```

결과

```text
/bin/bash
```

### 왜 이렇게 되는가?

커널과 관계 X

로그인할 때 설정된 **환경 변수(Environment Variable)** 값을 출력하는 것뿐임 (현재 Shell과 다를 수 O)

---

## ③ 배포판 확인

```bash
cat /etc/os-release
```

결과

```text
NAME="Ubuntu"
VERSION="24.04 LTS"
```

### 왜 이렇게 되는가?

`/etc/os-release`는 배포판 정보를 저장한 파일

---

# 5️⃣ 꼭 기억할 것

✅ **Kernel**
- 하드웨어를 직접 제어
- 운영체제의 핵심

✅ **Shell**
- 사용자의 명령을 해석
- Kernel에게 작업을 요청

✅ **Distribution**
- Kernel 하나만 있는 것이 아니라
- 다양한 프로그램을 함께 묶은 완성된 운영체제


# 💡 한 줄 요약

> **사용자는 Shell에 명령을 입력하고, Shell은 Kernel에게 작업을 요청하며, Kernel은 Hardware를 직접 제어한다.**
